1서버 · 배포 상태

배포 사이트가 제대로 떠 있는지 확인

  • https://kog-mss.vercel.app/login 접속 필수로그인 화면이 정상 노출되는지
  • Vercel Deployments에서 최신 배포 ✓ Ready 상태 필수
  • 로그인 화면에 "Google 계정으로 로그인" 버튼 표시 필수
  • 로그인 화면에 "샘플 데이터로 체험하기" 노란 버튼 표시
  • 로고 · KOG INTERNATIONAL 텍스트 · 디자인 정상

2Google 로그인 흐름

실제 등록된 계정으로 로그인 테스트

  • 본인 Gmail로 로그인 → 대시보드 정상 진입 필수3초 이내 진입되는지 (속도)
  • 사이드바 하단에 본인 이름 + 직책 표시
  • 로그아웃 클릭 → 로그인 화면 복귀 중요
  • 다른 Gmail (외부 계정)로 로그인 시도 → "등록되지 않은 계정" 에러 필수화이트리스트 동작 확인 — 이게 안 되면 보안 문제
  • "샘플 데이터로 체험하기" → 데모 모드 진입 (사이드바에 SAMPLE 뱃지)

3대시보드 화면

발표에서 처음 보여줄 화면 — 가장 중요

  • 상단 4개 KPI 카드 정상 (지연 프로젝트 / 검사 임박 / 교육 / 공문서) 필수
  • 📝 오늘 업무일지 섹션 표시 (빈 상태면 "+ 업무 등록하기" 링크) 필수
  • 📅 이달 근태 달력 표시 — 오늘 날짜 에메랄드 색 강조 필수
  • 근태 색상 범례 (연차/반차/외근/출장/재택/병가) 보임
  • 우측 상단 "미읽은 알림 N건" 표시
  • 하단 "부서별 이달 업무 달성률" + "최근 알림" 섹션
  • "전체보기 →" 링크 클릭 시 정상 이동 중요

4업무일지 (핵심 데모 기능)

매일 사용할 가장 중요한 기능

  • 사이드바 "업무관리 → 업무일지" 진입 필수
  • 날짜 이동 (← →) 정상 작동
  • + 업무 등록 버튼 → 폼 열림 → 등록 후 즉시 목록에 추가 필수정기업무/프로젝트/돌발업무 3종 선택
  • 동그라미 클릭 → 체크마크로 변경 (달성 처리)
  • 연필 아이콘 → 편집 모드 → 제목/유형/설명 수정 가능
  • 📊 보고서 버튼 → 당일 / 주간 / 월간 리포트 정상
  • 리포트 부서 드롭다운 → 부서별 필터 작동
  • "인쇄/PDF" 버튼 → 인쇄 미리보기 (결재란 포함)

5근태 관리 (핵심 데모 기능)

신청 → 승인 워크플로우 시연

  • 사이드바 "인사관리 → 근태 관리" 진입 필수
  • 월간 캘린더 정상 표시 (오늘 강조)
  • + 근태 신청 → 모달 열림 → 유형 선택 후 신청 필수
  • "연속 등록(다일)" 체크 → 시작/종료일 입력 → 평일 N일 계산 표시
  • 대기 상태 근태가 캘린더에 주황색 점선으로 표시
  • 관리자 계정에서 대기 건 → 승인 / 반려 버튼 동작 필수
  • 반려 시 사유 입력 모달 → 입력 후 처리
  • 직원별 월간 현황에 연차 잔여일수 표시 (X/Y일)

6관리자 — 사용자 · 부서

admin 계정으로 로그인해서 확인

  • 사이드바에 "시스템 설정" 그룹 표시 (admin만 보임)
  • 사용자 관리 → 직원 12명 목록 정상 (이름/직책/부서/역할) 필수
  • "수정" 버튼 → 모달 → 이름/직책/부서/역할/입사일 수정 중요
  • "+ 사용자 추가" → 새 사용자 폼 (비밀번호 필드 없음)
  • 역할 변경 (admin/manager/staff) 인라인 작동
  • 활성/비활성 토글 작동
  • 부서 관리 → 6개 부서 표시 (또는 본인이 등록한 만큼)
  • 부서 수정/삭제 가능 (직원 있는 부서는 삭제 차단)

7인사 관리 메뉴들

조직 정보 화면

  • 직원 관리 → 12명 목록 표시, 입사일 NULL인 사람은 "-"로 표시 (56년 X) 필수최근 수정한 버그 — 잘 고쳐졌는지 확인
  • 조직도 → 부서별 카드 표시, 무소속 직원은 별도 섹션
  • 복지 관리 → HR 이벤트 목록 (없으면 빈 상태 안내)

8나머지 모듈 빠른 점검

각 페이지 진입만 확인 (에러 없으면 OK)

  • 프로젝트 → 목록 화면 정상 (등록한 프로젝트 없으면 빈 상태)
  • 공문서 → 목록 화면 정상
  • 교육 일정 → 목록 화면 정상
  • 정기검사 → 목록 화면 정상
  • 연간 일정 → 캘린더/목록 뷰 전환 가능
  • 알림 센터 → "알림이 없습니다" 또는 알림 목록 표시

9모바일 화면

발표 중 모바일에서도 보여주는 경우

  • 스마트폰 브라우저로 접속 → 자동 모바일 레이아웃 중요
  • 햄버거 메뉴(≡) 클릭 → 사이드바 정상 슬라이드
  • 대시보드 KPI 카드 2x2 그리드로 정상 표시
  • 모바일에서 Google 로그인 가능

10발표 환경 준비

실제 발표장 세팅

  • 노트북 풀 충전 + 충전기 챙김 필수
  • 프로젝터/모니터 연결 케이블 (HDMI/USB-C) 준비
  • 발표장 Wi-Fi 또는 LTE 핫스팟 준비 필수웹 시스템이라 인터넷 끊기면 데모 불가
  • 브라우저 북마크: 로그인 화면 + 대시보드 미리 열어두기
  • 발표자료 (PRESENTATION.html) 별도 탭에 미리 띄우기
  • 이 체크리스트도 별도 탭에 띄워서 진행 표시
  • 사용 안내 자료 (USER-GUIDE.html) 직원들에게 공유 준비
  • 시연용 더미 데이터 1~2건 미리 입력해두기 (빈 화면 방지)

11발표 시연 시나리오 리허설

실제 시연 흐름 한 번 연습

  • 시나리오 ①: 일반 직원 입장 — 로그인 → 대시보드 → 업무일지 등록 → 달성 처리 → 근태 신청
  • 시나리오 ②: 관리자 입장 — 로그아웃 → admin 계정 로그인 → 근태 승인 → 사용자 관리 → 보고서 출력
  • 각 시나리오 1~2분 내 시연 가능한지 시간 측정
  • 중간에 끊기는 부분 / 답변 어려운 질문 미리 정리
